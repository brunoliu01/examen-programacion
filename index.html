git init: funciona para crea un repositorio. ejemplo: al momento de crear un trabajo por primera vez utilizas el git init.

git clone: crea un repositorio dentro de un directorio. ejemplo: utilizas el git clone cuando quieres sacar información de algún repositorio.

git push: git push sirve para poder enviar al repositorio actual toda la información hecha u obtenida ejemplo: cuando terminas un trabajo o al menos un parte, después de poner el commit, lanzas el git push origin master.

git log : funciona para poder explorar el hisorial de commits. ejemplo: cuando quieres saber cómo y en qué momento utilizaste el commit, también para saber cómo están trabajando los demás, en caso sea en grupo.

git commit: se le añade a los archivos registrado, para poder verificar de forma mas específica el contenido pusheado. ejemplo: funciona en el caso de subir contenido al repositorio, para verificar cómo es que estás avanzando.

git status : verifica que carpetas se encuentra dentro del trabajo que estás realizando para ser comiteadas y pusheadas próximament. ejepmlo: si estás trabajando un proyecto con muchas carpetas y quieres verificar que estén todas dentro del comando para poder pushearlas.

git add: selecciona un archivo para que sea añadido cuando hagas un commit. ejemplo: cuando trabajas en un proyecto y deseas añadir las carpetas dentro del comando.

git remote: sirve para conectar un repositorio local a un repositorio remoto.

git rm: remueve un archivo.

git config: funcona para poder sincronizar la cuenta de git por primera vez.
